# 제2차 슈퍼로봇대전 가변 길이 한글 패치 구조

## v0.2.1-pre 메뉴 패치 구조

현재 시험판은 이전의 실행 중 arena 확장 방식을 사용하지 않습니다. 원본
`SECOND.WAR` 내부의 정적 여유 영역(`0x3C938..0x3CD58`)에 메뉴 스크립트와
문자열을 재배치하고, 원본 루트 블록(`0xE14..0x1231`)을 같은 실행 파일 안의
cave로 옮깁니다. 따라서 전투용 scratch와 UI 문자열이 겹치지 않습니다.

제2차 UI master는 77개 레코드, 스크립트 overlay는 339개 레코드와 727개
교체 span입니다. F7의 2바이트 인자와 compact/extended 상태를 구분하는
stateful 토크나이저를 사용하며, 미리보기 대화 렌더러의 F7 zero-argument
종료 규칙은 별도로 보존합니다. 메뉴·인터미션 진입 시 문자열 영역이
런타임 작업 버퍼에 덮어써지던 문제가 이 구조에서 수정되었습니다.

최종 빌드의 Track 1은 568,386,672바이트이며 SHA-256은
`e5d28d78005b08b5f1abbd7a381440c3f90438fc67657dafe62e26cb6b2206be`입니다.
빌더는 정적 repack, 포인터·숨은 참조 guard, ISO EDC/ECC, xdelta 역적용과
13개 회귀 테스트를 수행합니다. 에뮬레이터 실행은 자동화하지 않습니다.

이 빌드는 과거 고정 슬롯 번역을 사용하지 않는다. 기존 방식은 원래
레코드 길이를 넘으면 화자, 공백, 문장부호, 끝 문장을 차례로 지우고 남은
바이트를 잘랐다. 그 결과는 `second_full_translation_cache.rejected.json`과
`second_full_fixed_slot_rejected`로 격리했으며 생산 빌더에서 읽을 수 없다.

## 번역 대상

- `SECOND/2_SCE.BIN`: 시나리오 레코드 4,616개
- `BMESS2.BIN`: 런타임 그래프가 직접 참조하는 전투 레코드 17,364개
- `SECOND/2_DEAD.BIN`: start/end 슬롯이 가리키는 격추 레코드 95개
- 합계 22,075개 occurrence, 중복을 합친 번역 키 6,503개
- `SECOND/SECOND.WAR`: 프롤로그·조건 UI 91건, 음악·데모 제목 171건,
  메뉴 테이블 1,754건, 이름 1,248건, 제2차 전용 메뉴 77건과 공통
  메뉴 5건

BMESS의 단순 인용부호 스캔 결과 가운데 런타임에서 참조하지 않는 54개는
패치하지 않는다. 반대로 구조 파서가 확인한 누락 레코드 4개와 DEAD의
비인용 레코드 2개는 원장에 보강했다.

## 길이 확장

`2_SCE.BIN`은 51개 시나리오 블록의 102개 상대 포인터를 다시 계산한다.
텍스트를 직접 가리키는 `B1`, `B3`, `B4` 명령의 피연산자 1,051개도 새
레코드 위치로 옮기고 다음 블록을 4바이트 경계에 정렬한다.

`BMESS2.BIN`은 399개 CPE 블록의 type `0x10/0x11` leaf가 참조하는
문자열만 교체한다. 원문 문자열 공간을 안전한 hole로 회수한 뒤 확장
레코드를 재배치하고 모든 공유 leaf를 새 위치로 고친다. 각 CPE는 게임의
런타임 로드 슬롯인 `0x3000`바이트를 넘을 수 없다.

`2_DEAD.BIN`은 255개 start/end 쌍과 마지막 경계를 다시 만들며 빈 슬롯과
파일 끝의 미참조 11바이트를 그대로 보존한다.

`SECOND.WAR`의 원래 고정 슬롯에 한국어 메뉴를 잘라 넣지 않는다. 루트 VM
스크립트와 프롤로그 풀을 함께 옮기고, 음악·데모 제목, 메뉴 테이블,
파일럿·기체 이름과 인터미션 레코드를 실행 파일 내부의 정적 여유 영역으로
재배치한다. 원본 레코드 SHA-256, 포인터 대상과 제어 바이트를 확인한 뒤
참조 포인터를 새 RAM 주소로 바꾼다.

정적 pool은 파일 오프셋 `0x3C938..0x3CD58`에 있으며, 원본 루트 블록은
`0xE14..0x1231`에서 같은 파일의 cave로 옮긴다. 전투 화자 문자열용
4×512바이트 scratch는 기존 BSS에 남겨 두되 UI pool과 겹치지 않도록
보호한다. 따라서 실행 중 BIOS heap을 확장하거나 게임의 작업 영역을
재사용하지 않는다.

네 파일은 원래 ISO 할당 공간에 억지로 맞추지 않는다. 원본 볼륨이 예약한
후미 섹터에 새 MODE2/Form1 extent로 기록하고 ISO9660 디렉터리의 LBA와
크기를 양 endian으로 갱신한다.

## 표시 레이아웃과 폰트

번역 문장은 먼저 화자와 정상 띄어쓰기를 포함한 완전한 문장으로 확정한다.
그 뒤 빌더가 한 줄 26칸, 페이지당 3줄로 단어 경계 줄바꿈을 만든다. 한
페이지를 넘으면 `F7` 페이지 전환을 추가한다. 원문의 `F7`과 `F9` 동적 이름
제어는 원래 text part 사이 순서로 보존하며 어떤 글자도 잘라 내지 않는다.

기본 글꼴은 11픽셀 잉크 폭의 갈무리14 KS X 1001 한글 2,350자를 사용한다.
번역에 KS X 1001 밖의 글자가 필요하면 남은 `0xA2F~0xAFF` 슬롯에 필요한
글자만 추가 렌더링한다. 매핑되지 않은 문자는 버리지 않고 빌드를 실패시켜
검수자가 확인하게 한다.

## 필수 검증

최종 빌드는 다음 조건을 모두 통과해야 한다.

- 모든 배치에 독립 검수자와 승인 용어집 해시 기록
- 화자, 따옴표, 문장 끝, 정상 띄어쓰기, 일본어 잔존 자동 검사
- 51개 SCE 블록, 1,051개 직접 참조, 4바이트 정렬 유지
- 399개 BMESS CPE와 `0x3000` 런타임 상한 유지
- DEAD 255쌍, 95개 live 레코드, trailing 데이터 유지
- UI 3,608개 구조 레코드의 source guard와 포인터 대상 검사
- 메뉴 스크립트 339레코드·727 span의 제어 바이트 보존
- 메뉴 테이블 1,754건과 이름 1,248건의 전체 coverage 검사
- 전투 scratch 최대 366바이트가 512바이트 슬롯 안에 있고 정적 UI pool과
  겹치지 않는지 검사
- 새 ISO extent 내용과 디렉터리 LBA/크기 일치
- 변경된 모든 MODE2/Form1 섹터의 EDC/ECC 재계산 검증
- xdelta를 원본 Track 1에 역적용한 결과가 패치 Track 1과 SHA-256 일치

에뮬레이터 화면 확인은 사용자가 직접 수행하므로 빌드 과정에서 에뮬레이터를
자동 실행하지 않는다.
