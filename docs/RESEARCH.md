# 조사 기록

## 확인한 글꼴 경로

게임 실행 파일 조사 결과 일본어 글꼴은 디스크에 독립된 글꼴 이미지로 저장된 것이 아니라 일본판 PlayStation BIOS 서비스 `B0:51` (`Krom2RawAdd`)를 통해 참조됩니다. 따라서 첫 표시 시험은 게임 디스크가 아니라 BIOS KROM의 복사본을 대상으로 진행했습니다.

일본판 BIOS KROM은 파일 오프셋 `0x66000`에서 시작합니다. 조사한 배열은 비한자 524자 뒤에 1수준 한자 2,965자가 이어지며, 글자 하나는 `16×15` 1bpp 비트맵을 MSB 우선 2바이트 × 15행으로 저장한 30바이트 셀입니다.

## 한글 시험 배열

KS X 1001 완성형 한글 2,350자(EUC-KR `B0A1–C8FE`)를 순서대로 생성해 첫 2,350개 1수준 한자 슬롯(Shift-JIS `889F–94FC`)에 넣었습니다.

```text
첫 글자 오프셋: 0x69D68
마지막 셀 시작: 0x7B0AE
교체 영역 끝:   0x7B0CB
셀 수:          2,350
영역 크기:      70,500 bytes
```

이 방식은 문자 표시 경로와 글꼴 크기를 확인하기 위한 임시 실험입니다. 기존 Shift-JIS 문자열을 한글 인코딩으로 해석하게 만들지는 않으므로, 다음 단계에는 게임 문자열 포맷과 실행 파일의 문자 처리 루틴을 별도로 수정해야 합니다.

## 재현 도구

- `tools/extract_psx_iso.py`: MODE2/2352 raw BIN의 ISO9660 파일 조사용
- `tools/dump_ps1_krom.py`: 본인이 보유한 일본판 BIOS KROM의 PNG 덤프용
- `tools/build_hangul_krom.py`: Galmuri14 BDF에서 한글 BIN·PNG·대응표 생성용

원본 게임 파일과 BIOS에서 추출한 결과물은 저장소에 포함하지 않습니다.
