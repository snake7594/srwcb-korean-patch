# v0.9.1 — 제3차 메뉴·이름까지 전체 한글화 (v0.9.0 표시 깨짐 수정)

**v0.9.0의 "메뉴가 깨져 보이는" 문제를 근본 해결한 릴리스입니다.** 제3차의 대사뿐
아니라 **메뉴·이름·시나리오 제목·맵 UI까지 전부** 한국어로 바뀝니다.
v0.9.0은 프리릴리스로 내렸으니 이 버전을 사용하세요.

## v0.9.0에서 무엇이 문제였나

한글 폰트는 원본 글꼴의 **한자 글리프 자리(인덱스 0x101~)** 를 덮어씁니다. 번역된
대사는 새 한글 인덱스로 다시 인코딩되므로 정상이지만, **번역하지 않은 메뉴·이름의
한자**는 그 자리에 한글이 들어와 있어 깨진 자모로 보였습니다(카타카나·영문은 보존
영역이라 일본어로 정상 표시). 즉 대사만 번역해서는 구조적으로 해결되지 않고,
**화면에 나오는 모든 텍스트를 번역해야** 합니다. 제2차가 멀쩡했던 이유도 이것입니다.

## v0.9.1에서 한 일 — 제3차 실행파일 UI 전면 한글화

- **이름·데이터 테이블 10종 3,000여 레코드**: 시나리오 제목, 파일럿 약칭/정식명,
  기체명, 무기명(1,408), 정신 커맨드, 지형, 파일럿 특수기능, 유닛 능력, 강화 파츠
- **시스템 메뉴(ui_master) 380개 치환 구간**: 시스템 설정·이동 범위·사운드·전투 BGM
  설정·버튼 설정·부대 목록 등
- **맵 UI 라벨 356개**(바이트-정확 in-place): 반격명령·페이즈 종료·부대표·작전 목적·
  정신 검색·턴 수·자금 등 맵 화면 상시 표시 항목
- **음악/데모 제목 162개**
- 제2차 번역 자산을 일본어 원문 기준으로 재사용(2,100여 건)하고, 제3차 고유 항목
  약 100건(시나리오 제목 57 등)을 신규 번역 — 제2차 표기 관례(솔로몬/액시즈/빅 잠/
  컴배틀러 V/정예병 등)를 그대로 따랐습니다.

### 저장 공간 확보 방식

한글은 원문보다 커서 테이블이 넘칩니다. 실행파일을 **늘리지 않고**, 최종 한국어
텍스트가 한 번도 참조하지 않는 **미사용 글리프 슬롯**을 도너 영역으로 회수해
넘치는 레코드를 배치하고 포인터를 재조준했습니다(제2차와 동일한 기법).
사용량 6,504 / 가용 46,720 바이트.

## 검증

- UI 주입 후 원본 일본어 잔존 확인: 반격명령·페이즈종료·부대표·작전목적·정신검색·
  턴수·자금·출격유닛선택·행동종료 등 **모두 0건**
- 단독판 이식본과 컴플리트 박스판의 테이블 디코딩 대조: **표시 대상 3,200/3,200 일치**
  (나머지 24건은 게임이 표시하지 않는 테이블 경계 바깥 영역)
- 대사 주입 표본 검증, 이미지 read-back SHA-256 일치, xdelta 역적용 라운드트립 일치
- 배틀 스크래치 재계산: 필요 0x16A / 확보 0x200
- 처음부터 끝까지의 실기 완주 검증은 미완료입니다.
