# 제3차 실행파일 UI 한글 주입

v0.9.0의 "메뉴 깨짐"을 해결한 도구. 한글 폰트가 원본 **한자 글리프 자리**를 덮어쓰므로,
화면에 나오는 텍스트를 **전부** 번역해야 깨짐이 사라진다(제2차와 동일한 이유).

- `build_third_full.py` — 마스터 빌드. ①모든 한국어에서 최종 extra 글리프 산출
  ②`build_dynamic_font` ③대사(3_SCE/BMESS3/3_DEAD) 재빌드 ④임베디드 BMESS3 테이블 +
  배틀 스크래치 패치. **순서 중요**(폰트 먼저, UI 주입은 마지막 — 도너가 폰트 영역 안).
- `inject_third_ui.py` — UI 주입. 코덱 토큰 단위로 걸으며 최장일치 바이트 스팬을 한글로
  치환(제어 인자 오염 방지). 대상: 이름/데이터 테이블 10종, ui_master, 음악·데모 풀,
  맵 라벨 힙(제2차 오프셋 +0x2dc, 바이트-정확 in-place).
- `third_ui_translations.json` — 제3차 고유 신규 번역(시나리오 제목 등).

**도너 공간**: 최종 한국어가 한 번도 참조하지 않는 글리프 슬롯을 회수해 사용
(0x000-0x100 보존 영역은 항상 유지 — 미번역 라틴/카타카나가 계속 렌더된다).
실행파일 크기는 늘리지 않는다.

**테이블 규칙**: 레코드가 **자기 풀 안에 완전히** 들어있는 것만 재패킹한다. 경계를 넘는
레코드는 공유 레코드이므로 건드리면 다음 테이블의 포인터 테이블이 깨진다.
